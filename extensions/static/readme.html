Redoist

The Redoist integration for Todoist allows a user to create copies of tasks that are continually updated. Tasks can be linked within the same project or across different projects. Links can also be one-way or bidirectional.

Creating a Redoist Task

First select a task you want to create a copy of. Open the context menu of the task and select the Redoist integration. This will open a window where you will choose the project the new task will be created in and the type of link you want to create.

There are three options for the type of link:

  • Outbound
  • Inbound
  • Bidirectional

Outbound links will use the existing task as the source for any updates. When the source task is updated, the linked task will be updated to match. Inbound links will instead use the new task as the source for any updates. Bidirectional links will update both tasks to match each other.

Identifying Redoist Tasks

Redoist will add labels to tasks to indicate that they are linked and how. For one-way links, the label will be redoist:source or redoist:destination, indicating which task will be used to update the other. For bidirectional links, the label will be redoist:bidirectional.

Updating Redoist Tasks

When a task is updated, Redoist will update any linked tasks to match. This includes changes to the following information:

  • Title
  • Description
  • Due Date
  • Priority
  • Labels
  • Parent Task (varies)
  • Comments
  • Completion
  • Deletion

The assignee of a task will not be updated or assigned by the Redoist integration. This is to prevent unintended access of users to projects of which they aren't a member.

When creating a Redoist task from a subtask, if the parent task is also linked by Redoist, then the new subtask will be created as a subtask of the corresponding linked parent task. If the parent task is not linked, then the new subtask will be created as a top-level task.

Removing Redoist Links

There are three ways to remove the linkage between Redoist tasks:

  • Delete the source task
  • Complete the source task
  • Use the integration on either linked task and select the Unlink option

Completion and deletion of a task will remove the linkage between tasks only after the linked task has been closed or deleted. Completing or deleting a destination task will not affect the source task and may cause errors.
